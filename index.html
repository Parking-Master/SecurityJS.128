How it works

Security.js uses a dynamic-database ("Cookies / Storage") to store information like SSL approvement, site verification, etc. How does it do this?

Imagine math... Ok so you add up +1 (100) times. It's easy, but it takes a while.

With Security.js, the library counts up every little piece of site-information. It's not something you see in action, but it happens behind the scenes.
